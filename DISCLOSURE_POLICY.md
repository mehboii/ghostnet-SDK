# Disclosure Policy

## Private Disclosure

All security vulnerabilities in `@n11x/ghostnet-sdk` must be reported privately:

**Email:** security@n11x.dev
**Subject:** `[SECURITY] @n11x/ghostnet-sdk — <brief description>`

**Do NOT open public GitHub issues for security vulnerabilities.**

## What to Include

- Description of the vulnerability
- Steps to reproduce
- Affected versions
- Potential impact assessment

## Timeline

| Stage | Target |
|-------|--------|
| Acknowledgment | 48 hours |
| Initial assessment | 7 days |
| Fix development | 30 days |
| Public disclosure | 90 days from initial report |

We follow a **90-day coordinated disclosure window**. If a fix ships before 90 days, public disclosure may happen sooner with mutual agreement between the reporter and maintainers.

## Scope

This policy covers the `@n11x/ghostnet-sdk` npm package and its published artifacts. For relay infrastructure issues, contact the same email.

## Recognition

Reporters will be credited in release notes unless anonymity is requested.
